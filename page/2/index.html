https协议

https的速度回比http慢, 但是安全

对称加密与非对称加密

对称加密是采用想用的密钥来加解密, 优点是速度比较快, 缺点是和多人交流的时候, 需要管理多个密钥。
非对称加密是加解密采用公钥和私钥, 公钥加密只有私钥解密, 私钥加密只能公钥解密, 这样可以验证身份, 可以保证机密性,
但是效率慢, https协议采用了两者结合

https

https是在应用层http协议到传输层tcp协议之间添加一个ssl层, 当客户端在请求服务器的时候, 首先是tcp的三次握手

  • 客户端向服务器发送自己支持的非对称加密算法
  • 服务器向客户端发送自己的证书, 这个证书是如何由来的呢
  • 接下来客户端会对这个证书做一下验证, 证书是否过期等
  • 请求证书的颁发机构的公钥, 对证书的签名进行解密, 解密成功说明证书没问题, 是我们要访问的目标网站
  • 然后对证书hash值是否等于签名。一切都没问题了, 非对称加密的部分就结束了
  • 上面的步骤得到目标网站的公钥(P), 开始交换对称秘钥了, 客户端随机生成一个秘钥, 用P加密, 发给服务器
  • 服务器用自己的私钥解密, 拿到里面的对称加密的秘钥(S), 这样对称加密的秘钥就商量好了, 中间看到的都是密文, 不会泄露这个秘钥, 接下来的数据来回, 都要用秘钥S来加解密

其他

  • https每次请求都要经过一些对称加密秘钥的商量

GO并发机制

GoLang在操作系统提供的内核线程上，搭建了自己的两级线程模型，叫goroutine，它不同于线程，进程，协程。 GoLang不推荐共享内存的方式传递数据，我们都知道在进程间通信的几种方式中，共享内存的方式是最高效的，这里不推荐主要是从编程的复杂度来考虑的，就像陈硕的他的mudo网络库里面提到的，进程间通信，他推荐用socket方式，原因雷同，考虑的编程方便和日后的扩展，效率损失其实可以忽略不计的

一.GO线程模型的实现

goroutine可以看做GoLang应用层线程，实现主要有3个核心元素

M: 一个M代表一个内核线程
P: 一个P代表一个Go代码片段所需要的资源
G: 一个G代表一个Go代码片段

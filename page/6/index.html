php内核zend_string

额… ,我用简单的代码，表达PHP内核是如何处理字符串的

struct _zend_string这个结构体,包含三个变量,分别介绍:

int gc 引用计数统计
int hash 该字符串的hash值
int len 字符串的长度, 二进制安全的, 不会出现控制符就结束字符串
char val[1] 一个长度的字符数组

咋一看_zend_string没有保存字符串的地方, 其实,初始化方法(initString)中, 在申请内存的时候会 + 字符串的长度, 然后,把字符串存在这个多申请的内存里面,不错其实位置要往前挪一个字节,挪到_zend_string var的位置, 最后空出来的一个字节存空字符(\0)

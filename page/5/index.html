webbench源码分析(一)

一. 简介

  webbench通过fork出多个子进程，每个子进程都循环做web访问测试。子进程把访问的结果通过pipe告诉父进程，父进程做最终的统计结果。

百度百科
github.com

二.安装

centos为例:

wget http://blog.s135.com/soft/linux/webbench/webbench-1.5.tar.gz
tar zxvf webbench-1.5.tar.gz
cd webbench-1.5
make && make install

三.使用方法

5个并发(线程), 一直请求百度首页, 持续时间10s

压测结果:
  每分钟可以请求552次，每秒钟传输1080153字节(这两个数据是根据本次压测结果推算得出的),10秒内请求成功92次，失败0次

四.实现原理

  当程序启动的时候，程序首先解析用户传过来的参数，比如 -c 这个就是并发数, 用来决定创建的线程数, 并发数等于子线程数, 在每个子线程刚开始均会设置一个定时器(时长根据 -t ), 子进程循环请求目标url，然后接收返回数据，每次请求的结果情况，要通过管道发送给父进程， 到点后，子进程都退出，父进程检查到子进程都退出了，也跳出循环，计算一下数据，显示给用户。
